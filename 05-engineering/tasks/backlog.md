## retrieval-engine
Discipline: backend
Status: done
Depends on: corpus-ingestion, metaculus-ingestion
Context: spec.md §src/retrieval/retriever.py; three modes: embedding, hybrid, metadata; RunConfig drives behavior

### What to build
`src/retrieval/retriever.py`:
- `retrieve_analogues(question, config, chroma_client, session) -> list[Analogue]`
- `embedding` mode: ChromaDB cosine query on question text → top-k
- `metadata` mode: SQL filter on event_type + region, rank by date proximity to question resolution_date
- `hybrid` mode: ChromaDB query → score each result with metadata score → weighted re-rank → top-k
- `Analogue` dataclass: `event: HistoricalEvent, similarity_score: float, features_used: dict`

### Done when
- Given any question from the DB, `retrieve_analogues` returns exactly `config.top_k` results in all three modes
- Hybrid mode results differ from pure embedding results (demonstrates re-ranking is active)
- Unit test: deterministic retrieval given same question + config

## scoring
Discipline: backend
Status: done
Depends on: synthesis-engine
Context: spec.md §src/scoring/scorer.py; Brier score formula; community baseline comparison

### What to build
`src/scoring/scorer.py`:
- `score_prediction(prediction, question) -> ScoreResult`
- `brier_score = (probability_estimate - resolution_value) ** 2`
- `community_brier_score = (community_probability - resolution_value) ** 2`

`ScoreResult` dataclass: `brier_score: float, resolved_value: float, community_brier_score: float`

### Done when
- Perfect prediction (prob=1.0, resolved=1.0) → brier_score=0.0
- Random guess (prob=0.5, resolved=1.0) → brier_score=0.25
- Community baseline is computed and stored for every scored prediction

## offline-loop-runner
Discipline: backend
Status: done
Depends on: scoring
Context: spec.md §src/runner/offline_loop.py; full pipeline; tqdm; graceful errors; RunResult written to DB

### What to build
`src/runner/offline_loop.py`:
- `run_offline_loop(config, session, chroma_client, anthropic_client) -> RunResult`
- Fetches all resolved questions from DB (respects `config.max_questions` cap)
- Per question: retrieve → synthesize → score → persist Prediction + Score to DB
- tqdm progress bar with current mean Brier score displayed
- Catch + log per-question exceptions, continue
- Skip questions already predicted in this run
- On completion: write `RunResult` to DB with mean/median Brier, n_predictions, cost_usd

`scripts/run.py`:
- `python scripts/run.py --config experiments/v1.yaml [--dry-run] [--max-questions N]`
- Overrides from CLI flags take precedence over config file

`experiments/v1.yaml` — baseline config as specified in spec.md

### Done when
- `python scripts/run.py --config experiments/v1.yaml --dry-run --max-questions 5` completes and writes a RunResult row
- `python scripts/run.py --config experiments/v1.yaml --max-questions 20` generates 20 Prediction + Score rows
- Interrupting and re-running skips already-predicted questions in the same run
